# 《大学生健康作息管理系统》项目提案

**Stack Overflow小组**    王晨瑜 董弈齐 曾梓航

### 一、 项目立意与核心优势

针对熬夜、作息紊乱这一生活痛点，打造具备实用性的工具。

### 二、 完整核心功能设计

本项目重心在于数据分析与健康预警：
1. **全维度日常记录**：入睡时间、起床时间、午睡时长、当日运动时长、久坐时长等情况。
2. **“跨天睡眠”精准统计**：针对大学生常在0点后入睡的特点，重构物理日期逻辑（如将每日时间切分点设为早晨6:00），精准处理跨天睡眠数据，自动统计每日睡眠时长、一周规律及熬夜天数。
3. **熬夜健康风险预警**：提供连续熬夜天数预警、睡眠不足健康防线提醒、作息紊乱分析报告。

5. **趣味健康反馈**：生成健康作息改善建议与睡眠质量评分，引入带有趣味性的“修仙称号系统”（如连续3天熬夜触发“熬夜修仙者”警告等）。

附加功能： **学期作息趋势对比**：针对期中期末周一键生成作息分析报告，直观展现压力下的作息退化。

### 三、 核心架构设计与C++面向对象技术亮点

本项目在底层架构上将全面融合 Qt 框架与 C++ 高级特性。
1. **前端展示与通信**：利用 Qt 构建现代化 UI，实现界面操作与后台数据的单向解耦。
2. **纯 C++ 核心逻辑层 (Model) 与封装**：设计C++的 `ScheduleRecord` (作息记录类)、`HealthAnalyzer` (睡眠分析类)、`AlertSystem` (预警类)、`FileStorage` (持久化类)。
3. **使用继承、多态完成基本骨架搭建，模板编程 (Templates) 与 STL 容器**。
4. **异常处理机制 (Exception Handling) 与 UI 联动**：定义继承自 `std::exception` 的自定义异常类（如 `InvalidTimeException`、`DataCorruptionException`）。当探测到业务逻辑错误（例如：设定的入睡时间晚于起床时间）时，底层抛出异常，前端 View 层利用 `try-catch` 捕获后，触发对应的 Qt 警告弹窗（QMessageBox::critical），展现极强的软件健壮性。

### 四、 Qt 现代图形界面与全维度交互体验

1. **清晰的操作控制台**：
   - 引入 `QCalendarWidget` 构建左侧日历面板。用户用鼠标点击任意日期，右侧动态渲染当天的作息详细数据，摒弃命令行繁琐的键盘查询。
   - 使用 `QTimeEdit` / `QDateTimeEdit` 控件实现精准的入睡/起床时间手动选择与校准。

3. **便捷打卡与系统托盘守护**：
   - **主界面一键快捷打卡**：设计醒目的“准备入睡”和“我醒了”超大悬浮按钮，获取系统级时间戳一秒打卡。
   - **后台静默守护体验**：程序支持最小化隐藏至系统右下角任务栏托盘。
   - **桌面气泡提醒**：当后台检测到用户连续坐立超过 3 小时，或当前已跨越凌晨 2:00 仍未点击“入睡”按钮时，将从屏幕右下角直接弹出系统级气泡警告，提醒“请立刻休息！”，提升交互体验。
附加功能：**专业级数据可视化分析 (QtCharts模块接入)**：
   - 绘制“熬夜波动曲线”，更直观清晰，方便对比工作日/周末/期末周的作息差异。
